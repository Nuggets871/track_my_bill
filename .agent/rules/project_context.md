RULE PROJECT_CONTEXT

This project is a personal finance application focused on budget tracking, expenses, subscriptions, housing costs, and investments.
The goal of the project is clarity, maintainability, and long-term evolutivity rather than raw performance or over-engineering.
The application is designed to be understandable at a glance by a human developer.

STACK
Frontend is built with Angular.
UI is implemented using DaisyUI.
Backend is built with Laravel.
The development environment is macOS using MAMP for PHP, MySQL, and local server management.

PRACTICES
All code must strictly follow Clean Code principles.
KISS is mandatory at all times.
SOLID principles must be applied rigorously.

Simplicity is always preferred over cleverness or optimization.
Readable and understandable code is more important than performant or compact code.

Variable names, function names, class names, and file names must always be explicit and meaningful.
Abbreviations are forbidden unless they are universally understood.
Names must describe intent, not implementation details.

No comments are allowed anywhere in the codebase.
Comments must never be generated, added, or suggested.
The code itself must be self-explanatory through naming and structure alone.

No markdown documents must ever be generated.
No documentation files are allowed.
All explanations must live implicitly in the code design.

END RULE
